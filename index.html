PARABÉNS!! ESTE É O HTML PARA SUBIR COM DOCKER!

Explicações e Outras Dicas

  • docker build -t "nome_para_imagem" .

    • "-t": Para nomear a imagem | "nome_para_imagem": Você que define | "." Referente ao diretório atual


  • docker run --name "nome_para_container" --rm -d -p "porta_desejada":80 "nome_da_sua_imagem"

    • "nome_container": Você que define | "--rm": Para remover container automaticamente após parar execução

    "-d": Para executar em Background (Não impedindo o prompt)

    "-p": Mapeamento de portas | "porta_desejada": Você define. Local | "80": Porta da nginx. Container | "nome_da_sua_imagem": Você definiu anteriormente

    Ou use de forma mais simples e rápida sem o build: docker run -d -p 80:80 nginx
    Dessa forma, terá um HTML padrão da nginx
    O proprio docker vai procurar a imagem "nginx" no repositório

    E por fim, acesse: localhost, ou 127.0.0.1, ou seu IP, digitando no navegador
    Por padrão o navegador vai usar a porta 80
